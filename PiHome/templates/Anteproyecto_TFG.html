1.Objetivo

El fin con el que se ha concebido este proyecto es diseñar e implementar un sistema domestico de bajo coste de manera que cualquiera pueda imitarlo, empleando para ello tanto software como hardware libre, con sus correspondientes ventajas. Como idea principal, está el gestionar el acceso a un recinto, aunque podría ampliarse de modo que se gestione el estado de los enchufes, la temperatura de la calefacción, actuar sobre la iluminación o la posición de las persianas.





2.Como

Para ello se ha valorado emplear elementos como Raspberry Pi o Arduino como hardware fundamental del sistema. Para la comunicación entre los sistemas se pretende usar tecnologías tales como NFC, ZigBee o Latch, además los protocolos comunes disponibles en cualquier sistema operativo actual.

a)Raspberry Pi

Este micro computador supone el centro de cálculo del sistema. La gestión podría estar a cargo de un sistema operativo GNU-Linux como Raspbian, que es una distribución derivada de Debian de la que hay mucha información y cuenta con una gran comunidad de soporte detrás, además de ser ligera y ser la opción preferida por los desarrolladores que trabajan con este hardware. Albergará un servidor web para almacenar los registros del sistema que podrán ser mostrados bajo una interfaz web. Para la comunicación con los periféricos del sistema se empleará el protocolo ZigBee.

b)Arduino

Arduino es un micro-controlador que será empleado para gestionar los periféricos del sistema. Se comunicará con la placa R. Pi mediante ZigBee. Dependiendo de la complejidad del sistema empleado dispondrá además de otras interfaces. En la versión más básica, control de acceso a un recinto, dispondrá de un módulo NFC activo, con el que identificar al individuo que quiera acceder, además, esta placa actuará sobre un servo motor que hará las veces cerrojo.

c)NFC

Se propone emplear esta tecnología como medio de comunicación entre el usuario y el sistema. El usuario deberá poseer una tarjeta identificativa RFID, sería indistinto que fuese en formato físico o bien que estuviese albergada en un dispositivo electrónico como puede ser un smartphone.

d)ZigBee

Por su escaso consumo energético, así como la posibilidad de un modo "StandBy", además de la sencillez y economía de su arquitectura tanto física como lógica, se sugiere usar esta tecnología para la conexión entre los diversos módulos del sistema. Teniendo también en cuenta, que puede funcionar en la banda de 868 MHz, por lo que no invadiría las frecuencias de otros elementos comunes en el hogar como el WIFI, o por su posible configuración topológica de red mallada, que aportaría seguridad en las comunicaciones del sistema.

e)Latch

Para añadir una capa de seguridad al sistema, se puede usar Latch (no es el circuito electrónico biestable síncrono) para regular el acceso al sistema.



3.Funcionamiento

El sistema estará comandado por un computador Raspberry Pi, el cual, gestionará las acciones de las placas Arduino (en el caso de los cerrojos o las persianas) o los enchufes o luces inteligentes.

La Raspberry tendrá que estar conectada a un punto de acceso a la red de área local con visibilidad desde el exterior. Los dispositivos inteligentes, en general, requieren conexión Wi-Fi, por lo que el core del sistema ha de ser capaz de acceder a está red. También es necesaria visibilidad desde el exterior, para poder controlar el sistema de manera remota mediante el uso del protocolo SSH.

Será necesario que esté implementada una interfaz web que permita mostrar los datos del servidor, en el que se recogerán los registros de cada interacción con el sistema o los usuarios.

En las partes en las que se requiera actuar sobre un mecanismo mecánico, esto se realizará a través de las placas de micro-controladores Arduino, que se conectarán mediante el conjunto de protocolos, ZigBee tanto entre ellos, formando una red sensorial, como de manera directa con el dispositivo Raspberry Pi.

Para que el sistema nos reconozca y así permitirnos el acceso al recinto, será necesaria la interacción, por medio del protocolo NFC. Un dispositivo NFC activo leerá la identificación del usuario contenida en una etiqueta RFID y lo mandará al servidor usando la interfaz 802.15.4 del dispositivo Arduino al que esté asociado.

Acceso al recinto

El comportamiento fundamental del sistema consistirá en: mediante un lector de tarjetas NFC, reconocer la identidad de un usuario que posea una tarjeta RFID o una aplicación equivalente. Dado el reducido tamaño de estos sistemas, el lector se puede instalar bajo la cubierta del interruptor del timbre, del cual podría recibir la alimentación, ademas de tener una batería para emergencias.

Cuando el sensor perciba la presencia de una tarjeta RFID, sacará del StandBy a la antena ZigBee y esta mandará el identificador al sistema central, gobernado por la Raspberry Pi, que contrastará que el usuario tenga permiso de acceso y registrará la petición. Si el resultado es afirmativo, se emitirá la orden al micro-controlador para que actúe sobre el servo-motor que abra el cerrojo para permitir el acceso. En caso contrario, no habrá ninguna respuesta.

El sistema registrará el acceso, en caso de producirse, el momento en que se accedió al recinto y durante cuanto tiempo estuvo la puerta abierta, para emitir un aviso en caso de olvido.
